优缺点

缺点

  • 当数组打到临界值，就会进行数组扩容+元素拷贝。数组进行这种操作相对慢一点
  • 在数组中间加一个元素，会导致数组元素全部往后位移

  • 数组实现随机读的效率很高

HashMap

数据结构

JDK 1.8以后，hashmap的数据结构是，数组 + 链表 + 红黑树
在最开始的时候，是存入数组，但是hash冲突之后，就会挂成链表，然后链表长度大于8以后，就将链表转为红黑树。

核心方法

核心参数

  • DEFAULT_INITIAL_CAPACITY

数组默认初始大小为16

  • DEFAULT_LOAD_FACTOR

负载因子为0.75，当元素个数达到数组容量*负载因子的时候，就会进行扩容。

  • Node

内部节点的数据结构

  • table

存储的数组

  • size

数组的大小

  • threshold

临界值。capacity * load factor
